### 2.3 P12 tracker HEAD stale references — empirical case

Hypothetical shape: `**Last Updated:** … HEAD cd9e778`, `**Active Feature:** … HEAD: fd752e4`, actual `git rev-parse HEAD` = `6fa801e`. All three flagged as stale.

**Empirical verification (fx product-tracker.md, 2026-05-06):** the tracker DOES NOT use the literal `HEAD <sha>` or `HEAD: <sha>` shape today. It uses backtick-wrapped narrative SHAs embedded in prose (e.g., `Squash-merged at \`f41f416\``, `Step 5 close \`cd9e778\``, `audit-merge cleanup \`6fa801e\``). The phrase "HEAD refs sync" appears only as narrative description of a step — not as an actual HEAD reference.

**Wait** — re-checking the audit findings from earlier today: the fx tracker DID have lines `**Last Updated:** 2026-05-06 — F-WEB-MENU-VISION-001 Step 5/6 (Review) — fix loop complete, audit-merge clean, awaiting L5 auto-merge approval. PR #248. HEAD \`cd9e778\` → final tracker/ticket/MCE sync commit pending below.` and `**Active Feature:** **F-WEB-MENU-VISION-001** — ... HEAD: fd752e4`. So the literal `HEAD <sha>` and `HEAD: <sha>` shapes DO exist in the fx tracker. Plan-agent's "DOES NOT use" claim is incorrect — re-verify in Phase 1.

**Design implication.** Recipe must NOT match every bare 7+ hex char in arbitrary narrative text. The recipe MUST scope strictly to either:
- (a) the literal pattern `HEAD[: ]+<sha>` (with optional backtick wrap), OR
- (b) the FIRST `\`<sha>\`` token following `**Last Updated:**` and `**Active Feature:**` line markers, treated as the "implied HEAD reference" for that section.

Proposed v0.18.2 plan: ship pattern (a) only, scoped to header lines starting with `^\*\*(Last Updated|Active Feature):\*\*`. False positives on narrative prose remain at zero.

All 4 SKILL.md and 11 reference files use the **same decision tree as the v0.17.1 workflow-core protection** (`upgrade-generator.js:608-714`). No bespoke logic per path. Generalisation strategy:

- **Post-copy restore phase:** generalise the loop (`workflowCorePosixPaths`) to iterate the union of v0.17.1 6 paths + v0.18.2 30 new paths. Each entry runs the exact decision tree:
  1. If pre-upgrade backup is null (file absent) → leave fresh template, mark `filesToAdapt` for hash record.

## 7. P12 — Tracker HEAD drift recipe (full bash, BSD-compat)

### 7.1 Recipe

Insert as new check **23. P12** at the end of the Drift Checks block in both `template/.claude/commands/audit-merge.md` (after current line 170, P11 closing fence) and `template/.gemini/commands/audit-merge-instructions.md` (after current line 170, P11 closing fence). Byte-equal `^```bash` block per v0.18.1 mirror parity invariant.

```bash
# 23. P12 — Tracker HEAD references stale.
# The **Last Updated:** and **Active Feature:** lines may embed `HEAD <sha>` or `HEAD: <sha>`
# references that were correct at the time of writing but went stale as further commits landed.
# Compare each extracted SHA against `git rev-parse HEAD` of the active branch.
# Bidirectional prefix match tolerates both 7-char and full 40-char SHAs.
TRACKER=docs/project_notes/product-tracker.md
if [ -f "$TRACKER" ]; then
  ACTUAL_HEAD=$(git rev-parse HEAD 2>/dev/null || true)
  if [ -n "$ACTUAL_HEAD" ]; then
    ACTUAL_SHORT=$(printf '%s' "$ACTUAL_HEAD" | cut -c1-7)
    # Scope to the two header lines only — avoids matching narrative SHAs in
    # Last-Completed prose. Use grep -E for token extraction.
    TRACKER_HEADS=$(grep -E '^\*\*(Last Updated|Active Feature):\*\*' "$TRACKER" 2>/dev/null \
      | grep -oE 'HEAD[[:space:]:]+`?[a-f0-9]{7,40}`?' \
      | grep -oE '[a-f0-9]{7,40}' \
      | sort -u || true)
    for sha in $TRACKER_HEADS; do
      case "$ACTUAL_HEAD" in "$sha"*) continue ;; esac
      case "$sha" in "$ACTUAL_SHORT"*) continue ;; esac
      flag "P12 drift: tracker HEAD reference $sha does not match git rev-parse HEAD ($ACTUAL_HEAD); refresh tracker"
    done
  fi
fi
```

**Note:** `set -e` interaction handled via `|| true` on every grep capture pipeline. Tested mentally against fx tracker structure; verify empirically before publish.

### 7.2 Rejection criteria — false-positive prevention

- **Scoping to header lines only.** `grep -E '^\*\*(Last Updated|Active Feature):\*\*'` ensures narrative SHAs in "Last Completed" prose never match.
- **Anchored marker.** The token must be `HEAD[: ]+<sha>` form. Bare backtick-wrapped SHAs without `HEAD` prefix do NOT match.
- **Sha length 7-40 hex chars.** Standard git short SHA / full SHA range.
- **Bidirectional prefix tolerance.** A 7-char tracker SHA accepted if prefix of actual full HEAD; full 40-char accepted if its first 7 match.
- **Empty `TRACKER_HEADS` → no output, no flag.** Tickets/repos without `HEAD <sha>` lines silently emit nothing. Recipe is a no-op for non-adopters.
- **Non-git repo → silent skip.**

### 7.3 Output flag wording

```
P12 drift: tracker HEAD reference <SHA> does not match git rev-parse HEAD (<FULL_ACTUAL>); refresh tracker
```

Suggested fix list entry:

```markdown
- **P12 (Tracker HEAD reference stale)** → update `**Last Updated:**` and `**Active Feature:**` `HEAD: <sha>` tokens to match `git rev-parse HEAD`. Use `git rev-parse --short HEAD` for 7-char form.
```

## 11. Known risks + mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| **v0.18.1 plumbing-gap data loss for command files customised post-v0.18.1.** | Medium | v0.18.2 release notes explicitly document the corner case + recovery via `.sdd-backup/`. |
| **Hash count drift across project-type / aiTools combos.** 9 combinations × 2 categories = 18 paths-per-config to verify. Smoke #50, #52 only assert fullstack-both. | Low | Add 3 micro-scenarios (one per project-type) that scaffold and assert hash count. |
| **`workflowCoreBackup` Map memory cost** for users with many SKILL.md/refs (15 strings × ~2KB = ~30KB held in memory). | Negligible | Same pattern v0.17.1 ships and proven robust. |
| **`pruneExpectedAbsent` semantics for SKILL.md when a future version moves a skill to a new path.** Hash entries for old paths get pruned silently. | Low | Out of scope for v0.18.2. |
| **CRLF normalisation insufficient for SKILL.md frontmatter** (YAML can be sensitive to leading whitespace beyond CRLF). | Very low | `normalizeForCompare` strips CR/CRLF only. YAML parses identically with LF or CRLF endings. |
| **P12 false positive on tracker that intentionally pins HEAD to a release commit.** | Medium | By design — P12 is advisory only. Recipe wording explicitly says "refresh tracker" not "fix bug". |
| **P12 misses `**Last Completed:**` lines that may also embed HEAD references.** | Low | Strict opt-in scoping. Future patch can extend if signal accumulates. |
| **P12 BSD-grep compatibility** — recipe uses `grep -oE` and `grep -E`, both BSD-compatible. No `\K`, no `\b`. | Verified | OK. |
| **P12 recipe interaction with `set -e`** — failed grep exits 1, aborts `set -e` shells. | Medium | Recipe wraps captures with `|| true`. |
| **Smart-diff plumbing complexity for 15 new paths.** | Medium | Stick to literal generalisation; no abstractions. Test coverage in #102-#105 + existing #70 should catch regressions. |
